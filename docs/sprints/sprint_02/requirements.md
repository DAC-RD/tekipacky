# 要件定義（Sprint 02）

## 概要

Sprint 01 で持ち越しとなった「認証の導入」を実装する。
メールアドレスによるマジックリンク認証を導入し、アプリのデータをユーザーごとに管理する。

---

## ユーザーストーリー

- 初めてアクセスしたユーザーは、アプリの概要を説明するランディングページを見て、使ってみようと思ったらメールアドレスを入力してサインインできる
- メールアドレスにマジックリンクが届き、クリックするだけでログインが完了する（パスワード不要）
- ログイン後はダッシュボードが表示され、自分専用の行動・ご褒美メニューとポイント残高が管理できる
- 別のデバイスや別のブラウザで同じメールアドレスでサインインすると、同じデータにアクセスできる

---

## 機能要件

### ランディングページ（未認証時のトップページ）

- `/` にアクセスした未認証ユーザーに表示する
- アプリ名・キャッチコピー・概要説明を表示する
- 行動メニュー・ポイント獲得・ご褒美の3つの特徴を説明する
- 「始める」「サインインする」等のCTAボタンからサインインページへ遷移できる

### サインインページ

- パス: `/signin`
- メールアドレスを入力するフォームを表示する
- 「マジックリンクを送る」ボタンを押すと、入力したメールアドレスに認証メールが送信される
- 送信後は「メールを確認してください」画面 (`/signin/verify`) にリダイレクトする

### マジックリンク認証

- メールは Resend を使って送信する
- メール内のリンクをクリックすると、JWT セッションが発行されてログイン状態になる
- ログイン成功後は `/`（ダッシュボード）にリダイレクトする
- リンクには有効期限を設ける（Auth.js のデフォルト: 24時間）

### ダッシュボード（認証済み時のトップページ）

- `/` にアクセスした認証済みユーザーに表示する（Sprint 01 の実装内容をそのまま継続）
- 追加変更なし

### API ルートの保護

- `/api/auth/**` 以外の全 API ルートは認証必須とする
- 未認証リクエストには `401 Unauthorized` を返す
- 認証済みリクエストには、JWT セッション内のユーザーID を `x-user-id` ヘッダーとして付与する（既存ロジックと互換）

### サインアウト

- ダッシュボードからサインアウトできる（UI は Sprint 02 スコープ内で実装）
- サインアウト後はランディングページ (`/`) にリダイレクトする

---

## 非機能要件

- セッション管理は JWT（Cookie に保存、HttpOnly）
- パスワードを保存しない（マジックリンクのみ）
- 既存ユーザーデータへの影響を最小化する（`email` カラムは nullable で追加し後方互換を確保）
- 既存の 175 本のテストが引き続き通ること

---

## 使用技術・サービス

| 項目 | 内容 |
|------|------|
| 認証ライブラリ | Auth.js v5（next-auth@beta） |
| メール送信 | Resend（Auth.js 組み込みプロバイダー経由） |
| セッション戦略 | JWT |
| DB アダプター | @auth/prisma-adapter |

## 対象外（将来対応）

- Google / GitHub 等の OAuth ログイン
- パスワードによるサインイン
- メールアドレスの変更機能
- アカウント削除機能
