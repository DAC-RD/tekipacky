# 要件定義（Sprint 03）

## 概要

Sprint 02 で持ち越しとなった「アカウント管理機能」を実装する。
ログイン中のユーザーが設定ページからメールアドレスの変更とアカウントの削除を行えるようにする。

---

## ユーザーストーリー

- ログインユーザーは、設定ページで現在のメールアドレスを確認できる
- メールアドレスを変更したいユーザーは、新しいメールアドレスを入力して確認メールを送信できる
- 確認メール内のリンクをクリックすると、メールアドレスが更新され、新しいアドレスで再ログインするよう促される
- アカウントを削除したいユーザーは、確認を経てアカウントと全データを削除できる
- アカウント削除後はランディングページに戻り、データが完全に消去されていることが保証される

---

## 機能要件

### 設定ページ

- パス: `/settings`
- 認証済みユーザーのみアクセス可（未認証は `/signin` にリダイレクト）
- 以下のセクションを表示する:
  - メールアドレス変更セクション
  - アカウント削除セクション（デンジャーゾーン）
- ダッシュボードから設定ページへ遷移できるリンクを追加する

### メールアドレス変更

- 現在のメールアドレスを表示する
- 新しいメールアドレスを入力するフォームを表示する
- 「確認メールを送る」ボタンを押すと、新しいメールアドレスに確認メールが送信される
- 以下の場合はエラーを返す:
  - メールアドレスの形式が不正
  - 現在と同じメールアドレス
  - 既に別のアカウントで使用済みのメールアドレス
- 確認メール内のリンクをクリックするとメールアドレスが更新される
  - リンクには有効期限を設ける（24時間）
  - 更新後はセッションをサインアウトし、`/signin` にリダイレクトして再ログインを促す
  - リンクが無効（期限切れ・不正）の場合はエラーメッセージを表示する

### アカウント削除

- 「アカウントを削除」ボタンを押すと確認ダイアログを表示する
- ダイアログで「削除する」と入力することで削除を確定できる（誤操作防止）
- 削除対象: ユーザー本人のデータ（行動・ご褒美・完了ログ・ポイント・セッション）をすべて削除する
- 削除後はサインアウトし、ランディングページ（`/`）にリダイレクトする

---

## 非機能要件

- メール変更の確認リンクは既存の `VerificationToken` テーブルを流用し、スキーマ変更不要とする
- アカウント削除は既存の `onDelete: Cascade` 設定を活用し、`User` レコードを削除するだけで関連データがすべて削除される
- 既存の 237 本のテストが引き続き通ること

---

## 使用技術・サービス

| 項目 | 内容 |
|------|------|
| メール送信 | Resend（既存インストール済み） |
| トークン管理 | VerificationToken テーブル（既存）を流用 |
| データ削除 | Prisma `onDelete: Cascade`（既存スキーマ設定済み） |

---

## 対象外（将来対応）

- Google / GitHub 等の OAuth ログイン
- 過去のポイント推移・行動統計ページ
